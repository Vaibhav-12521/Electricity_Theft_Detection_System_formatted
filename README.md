## 🔍 Overview

Electricity theft (also known as non-technical loss or NTL) costs utilities an estimated **$96 billion annually** worldwide. This system uses a three-model ML ensemble to automatically identify fraudulent consumption patterns — from bypassed meters to late-night industrial theft — and flags them in real time through a web dashboard.

**Key capabilities:**
- **Single reading detection** — analyze one meter reading instantly via sliders
- **Batch detection** — upload a CSV of readings and get bulk results
- **Live simulation** — watch the system monitor a simulated meter in real time
- **Historical analysis** — visualize theft patterns across a full dataset
- **Ensemble scoring** — combines Random Forest, Isolation Forest, and One-Class SVM for robust detection

## ⚙️ How It Works

### Full Pipeline

```
Raw Meter Data
      │
      ▼
DataPreprocessor.create_features()
  ├─ Time features (hour, weekday, is_night)
  ├─ Rolling stats (24h mean, std, ratio)
  ├─ Electrical features (apparent_power, real_power)
  └─ Anomaly flags (low_consumption, voltage_anomaly)
      │
      ▼
StandardScaler.transform()  ← fitted on training data
      │
      ▼
┌─────────────────────────────────────────┐
│           ENSEMBLE VOTING               │
│                                         │
│  RandomForest (40%)  ──────────────┐    │
│  IsolationForest (30%) ───────────►│ ──► theft_score
│  OneClassSVM (30%)  ──────────────┘    │
└─────────────────────────────────────────┘
      │
      ▼
theft_score > 0.5 → THEFT DETECTED 🚨
theft_score ≤ 0.5 → Normal ✅
```

### Ensemble Scoring Formula

```python
theft_score = 0.4 * rf_probability
            + 0.3 * (isolation_forest_prediction == -1)
            + 0.3 * (oc_svm_prediction == -1)

is_theft = theft_score > 0.5
```

## 🤖 ML Models

### Random Forest Classifier (Weight: 40%)

- **Type:** Supervised classification
- **Config:** 200 estimators, `class_weight='balanced'`
- **Why:** Highest accuracy when labeled data is available. Balanced weights handle the 83%/17% class imbalance without resampling.
- **Output:** Probability score 0–1

### Isolation Forest (Weight: 30%)

- **Type:** Unsupervised anomaly detection
- **Config:** `contamination=0.15` (expects ~15% anomalies)
- **Why:** Detects novel theft patterns not seen during training. No dependency on labels.
- **Output:** +1 (normal) or -1 (anomaly)

### One-Class SVM (Weight: 30%)

- **Type:** Unsupervised novelty detection
- **Config:** `nu=0.15`, `kernel='rbf'`, `gamma='scale'`
- **Training:** Fitted **only on normal data** — learns a tight boundary around legitimate consumption
- **Why:** Very sensitive to deviations from normal; complements Isolation Forest
- **Output:** +1 (within normal boundary) or -1 (outside boundary)

## 🔬 Feature Engineering

The `DataPreprocessor.create_features()` method generates 17 features from raw meter readings:

### Time-Based Features (4)

| Feature | Type | Description |
|---|---|---|
| `hour` | int 0–23 | Hour of the day |
| `day_of_week` | int 0–6 | Monday=0, Sunday=6 |
| `is_weekend` | binary | 1 if Saturday or Sunday |
| `is_night` | binary | 1 if hour ≥ 22 or hour ≤ 6 |

### Consumption Pattern Features (4)

| Feature | Type | Description |
|---|---|---|
| `consumption_kwh` | float | Raw meter reading |
| `consumption_rolling_mean_24h` | float | 24-hour moving average |
| `consumption_rolling_std_24h` | float | 24-hour rolling standard deviation |
| `consumption_ratio` | float | Current / rolling mean (spike detector) |

### Electrical Features (5)

| Feature | Type | Description |
|---|---|---|
| `voltage` | float | Line voltage in Volts |
| `current` | float | Line current in Amperes |
| `power_factor` | float | Ratio of real to apparent power |
| `apparent_power` | float | voltage × current (VA) |
| `real_power` | float | apparent_power × power_factor (W) |

### Anomaly Indicator Flags (4)

| Feature | Type | Condition |
|---|---|---|
| `power_ratio` | float | real_power / apparent_power |
| `low_consumption` | binary | consumption_kwh < 0.5 kWh |
| `high_pf_variation` | binary | \|power_factor − 0.9\| > 0.2 |
| `voltage_anomaly` | binary | \|voltage − 230\| > 20 V |

## 🕵️ Theft Pattern Types

The system is trained to recognize three archetypes of electricity theft:

### 1. Night Theft
```
Pattern:  High consumption between 10 PM and 6 AM
          Normal or near-zero usage during the day
Signals:  is_night=1, consumption_ratio spike, voltage anomaly
Use case: Commercial/industrial theft when inspectors aren't working
```

### 2. Meter Bypass
```
Pattern:  Sudden sustained drop to near-zero readings (0.3 ± 0.1 kWh)
          Unusually high power factor (0.95) — electricity being used
          but not recorded
Signals:  low_consumption=1, high power_factor, IsolationForest anomaly
Use case: Physical bypass wire installed around the meter
```

### 3. Tampered Meter
```
Pattern:  Erratic, exponentially distributed consumption
          Low power factor (0.4–0.7) — meter mechanism disturbed
          Voltage often abnormal (210 ± 15 V)
Signals:  high_pf_variation=1, voltage_anomaly=1, OC-SVM boundary breach
Use case: Magnetic tampering or mechanical interference with meter disc
```

## 📡 API Reference

### `ElectricityTheftDetector`

Located in `src/detection_system.py`.

#### `detect_theft()`

Analyze a single meter reading.

```python
from src.detection_system import ElectricityTheftDetector

detector = ElectricityTheftDetector()

result = detector.detect_theft(
    consumption_kwh=0.3,   # float — energy in kWh
    voltage=225.0,          # float — volts
    current=1.29,           # float — amperes
    power_factor=0.65,      # float — 0.0 to 1.0
    hour=2,                 # int — 0 to 23 (optional, defaults to current time)
    day_of_week=1,          # int — 0=Mon, 6=Sun (optional)
    is_night=1              # int — 0 or 1 (optional)
)
```

**Returns:**
```python
{
    'is_theft': True,              # bool
    'theft_probability': 0.73,     # float 0.0–1.0
    'rf_prediction': 1,            # int 0=normal, 1=theft
    'iso_forest_anomaly': True,    # bool
    'oc_svm_anomaly': False,       # bool
    'consumption_kwh': 0.3,        # float (echoed back)
    'voltage': 225.0,              # float (echoed back)
    'power_factor': 0.65           # float (echoed back)
}
```

#### `batch_detect()`

Analyze a DataFrame of meter readings.

```python
import pandas as pd

df = pd.read_csv('data/full_dataset.csv')

# - 'theft_probability'  float 0.0–1.0
```

## 📊 Data Format

### Input CSV Schema

```csv
timestamp,consumption_kwh,voltage,current,power_factor,is_theft
2023-01-01 00:00:00,2.45,231.2,10.5,0.92,0
2023-01-01 01:00:00,0.28,219.8,1.2,0.61,1
...
```

| Column | Type | Description |
|---|---|---|
| `timestamp` | datetime | ISO format: `YYYY-MM-DD HH:MM:SS` |
| `consumption_kwh` | float | Energy consumed in kilowatt-hours |
| `voltage` | float | Line voltage in Volts |
| `current` | float | Line current in Amperes |
| `power_factor` | float | 0.0 to 1.0 |
| `is_theft` | int | Ground truth: 0 = normal, 1 = theft (optional for inference) |

### Normal Consumption Ranges

| Parameter | Normal Range | Theft Indicator |
|---|---|---|
| Consumption | 1.5 – 5.0 kWh | < 0.5 kWh or > 8.0 kWh |
| Voltage | 210 – 250 V | < 210 V or > 250 V |
| Current | 5 – 30 A | < 1 A with high apparent usage |
| Power Factor | 0.85 – 0.99 | < 0.70 (tampering) |

```python
# Detection threshold (default: 0.5)
is_theft = 1 if theft_score > 0.5 else 0  # Lower = more sensitive
```

## 🔧 Troubleshooting

### `ModuleNotFoundError: No module named 'src'`

The `src/` directory is missing an `__init__.py` file.

```bash
touch src/__init__.py
```

Then run again from the **project root directory**, not from inside `src/`.

---

### `FileNotFoundError: models/theft_detector.pkl`

The model has not been trained yet. Run steps in order:

```bash
python data_generation.py
python src/model_training.py
streamlit run app.py
```

---

### `FileNotFoundError: data/full_dataset.csv`

```bash
python data_generation.py
```

Run this from the project root directory.

---

### Dashboard freezes for 10 seconds on load

This is a **known bug** in the Live Dashboard tab. The blocking `for` loop runs 100 iterations × 0.1s sleep. Refresh the page once the loop completes, or skip Tab 1 for now. See [Known Issues](#-known-issues) for the fix.

---

### `MemoryError` during model training

OneClassSVM has O(n²) memory complexity. Reduce training data:

```python
# In data_generation.py, change:
generate_normal_consumption(n_samples=5000)  # was 10000
generate_theft_consumption(n_samples=1000)   # was 2000
```

---

### Uploaded CSV gives `KeyError`

Your CSV must contain these exact column names:
```
timestamp, consumption_kwh, voltage, current, power_factor
```

Optional but useful: `is_theft` (for ground truth comparison).

---

### Low detection accuracy on real data

The model is trained on **synthetic data**. Real-world meter readings will have different distributions. Retrain the model on your actual AMI data for best results.

## ⚠️ Known Issues

| # | Issue | Severity | Status |
|---|---|---|---|
| 1 | Live Dashboard blocking loop freezes UI for ~10s | 🔴 High | Open |
| 2 | `detect_theft()` uses hardcoded zeros for rolling stats — silent prediction error | 🔴 High | Open |
| 3 | All training data is synthetic — no real-world validation | 🟡 Medium | Open |
| 4 | `models/theft_detector.pkl` overwritten on each training run (no versioning) | 🟡 Medium | Open |
| 5 | No column validation on CSV upload — missing columns cause unhandled crash | 🟡 Medium | Open |
| 6 | `numpy==1.24.3` and `scikit-learn==1.3.2` have newer security-patched versions | 🟢 Low | Open |

### Fix for Issue #1 (Live Dashboard)

Replace the blocking loop in `app.py` with:

```python
# Replace the for loop in tab1 with:
if 'readings' not in st.session_state:
    st.session_state.readings = []

sim_consumption = 2.5 + np.random.normal(0, 0.5)
# ... generate other values ...
result = detector.detect_theft(sim_consumption, sim_voltage, sim_current, sim_pf)
st.session_state.readings.append(result)

# Display metrics
col1, col2 = st.columns(2)
with col1:
    st.metric("Status", "🚨 THEFT" if result['is_theft'] else "✅ Normal")
with col2:
    st.metric("Risk", f"{result['theft_probability']*100:.1f}%")

time.sleep(1)
st.rerun()
```
